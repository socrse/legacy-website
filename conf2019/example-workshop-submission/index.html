Example workshop submission

Title and abstract

Using Foo to create interactive analyses of Bar

Foo is a popular data analysis module which can be used to create interactive analyses of Bar. We used Foo successfully during a recent project. This gave us an insight into some useful tips and tricks that we now want to share. During this workshop we will use a virtual machine with Foo installed and learn how to create an interactive analysis of Bar. Starting from a simple Foo analysis, we will build up to an interactive and full-feature visualisation of Bar that will could be applied to a wide range of different disciplines.

Length

90 minutes

Audience

This workshop will be of interest to anyone who has to perform analyses similar to Bar, or who supports researchers performing such analyses, or who have heard about Foo and want to learn more. We will assume knowledge of python and Linux, but will not expect any knowledge of Foo, and will provide a introduction to Bar.

Technology

I will need a virtual machine with one core per attendee, or a single core VM per attendee. Each attendee will need 512 MB. The VM needs to be installed with Ubuntu 18.04, and I will provide a list of deb packages to install in the form of a text file in my GutHub repo by the 9th August.

We will need a static microphone by the lectern PC, and the ability to plug a Mac in to the projector.

Outcomes

Attendees will have a good knowledge of Foo, or Bar analyses, and enough information to enable them to start applying Foo to create interactive Bar analyses of their own. We will support this outcome by uploading the workshop VM specs and slides to the conference app / website before the conference, thereby providing a resource that will help others repeat and build on our work.

Accessibility

We will prepare slides and VM specs for the workshop that will be uploaded to the conference app to be shared all participants before the conference. We will make use of the conference-provided microphones for all speakers and roving microphones for the audience during questions. We will ensure all of our slides use large and legible fonts, and that we have checked our colour choices so that all figures are understandable for people who are colour-blind.

Promotion

We will upload the slides and VM specs to the conference app and website as discussed above and will promote these on twitter and on the dedicated RSE19 conference channel on the UKRSE slack to encourage attendance. We will provide an email address to allow the audience to pre-submit questions, and will get in contact with groups involved with Foo and Bar to let them know that we are presenting this workshop at RSE19.

Mentorship

This will be the first workshop we have organised so we would request mentorship. In particular we would like someone to help us check the VM works properly and review the slides in July.
